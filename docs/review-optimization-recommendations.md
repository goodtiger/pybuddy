# PyBuddy Review — 产品方案 + Stitch 设计优化建议

> Review 日期: 2026-05-20 | 基于竞品对比 + 行业最佳实践

---

## 🔴 严重缺失 (P0 — 必须在 MVP 前补充)

### 1. 缺少家长面板 (Parent Dashboard) 设计
**现状**: 产品方案提到了"家长控制面板"，但 Stitch 中没有任何家长相关的屏幕设计。
**竞品对标**: Rodocodo、HeyOtto、Code Ninjas 都有完善的家长面板。
**为什么重要**: 付费决策者是家长，不是孩子。家长看不到学习价值就不会续费。

**建议新增屏幕**:
- **家长总览页**: 孩子本周学习时间、完成课程数、技能掌握雷达图
- **学习报告**: 按周/月生成，"孩子本周学会了：循环、变量、条件判断"
- **屏幕时间控制**: 设置每日学习上限（家长焦虑点）
- **订阅管理**: 免费版→付费版转化漏斗

### 2. 缺少 Onboarding (首次体验) 设计
**现状**: 产品方案完全没有 onboarding 流程描述，Stitch 也没有注册/引导页面。
**竞品对标**: Duolingo ABC 用"无需注册→先上一课→注册保存进度"策略。
**洞察**: quizzy.earth 创始人说"8岁孩子对你的 onboarding 流程零耐心。对他们来说没有'开始前的准备'，只有好玩和不好玩。"

**建议新增屏幕**:
- **Splash + 立即开始**: 跳过注册，直接进入第一节课（60秒内到达 "Aha moment"）
- **角色扮演引导**: 吉祥物小海龟自我介绍，"跟我来！我教你写第一行代码！"
- **注册时机**: 完成第1课后才弹出注册（"保存你的作品需要一个小账号哦"）
- **家长端 onboarding**: 注册后引导家长设置屏幕时间、学习目标

### 3. 缺少错误/空状态 (Error/Empty State) 设计
**现状**: 产品方案只提到"友好错误提示"，但没有具体的错误页面设计。
**竞品对标**: Duolingo 用 Duo 吉祥物让 404 和空状态变得有趣。

**建议新增**:
- **代码错误页**: 小海龟挠头 + 友好提示（不是红字 SyntaxError）
- **空成就页**: "还没有徽章？完成3节课就能解锁第一个！"
- **空项目页**: "还没有作品？试试第1课画一个正方形！"
- **网络错误页**: "小海龟迷路了，检查下网络再试试"

### 4. 缺少加载/过渡动画 (Loading/Preloader)
**现状**: 没有考虑 Python 运行时的加载等待（Skulpt/Pyodide 都需要几秒）。
**竞品对标**: Duolingo 每次加载都有 Duo 动画 + 语言小知识。

**建议**:
- 加载时显示"Python小知识"卡片（利用等待时间教学）
- 小海龟动画进度条（跑步、游泳、睡觉）

## 🟡 重要优化 (P1 — MVP 后尽快补充)

### 5. 产品方案中课程结构问题

**问题 A: Level 1 从15课开始就用Turtle太早**
- 第11课就引入 `import turtle`，对零基础孩子跨度太大
- **建议**: Level 1 前10课纯积木（不涉及Turtle），第11-15课用积木控制Turtle

**问题 B: 缺少 "调试(Debugging)" 主题**
- AI4Kids 的 Think→Try→Reflect 是核心差异化
- Koda Academy 的 "Break It" 模式（故意留bug让孩子修）非常有价值
- **建议**: 在 Level 2 加入专门的 "Bug猎人" 课程，培养工程思维

**问题 C: 100课的学习曲线过陡**
- Level 3 就开始做"猜数字游戏"和"文字冒险游戏"，8岁孩子可能打字都困难
- **建议**: Level 3 仍以Turtle项目为主，游戏项目推迟到 Level 4

### 6. Stitch 设计一致性问题

**问题 A: 多个 Adventure Map 变体没有统一标准**
- Stitch 生成了4个不同的 Adventure Map，需要选择一个最优版本
- **建议**: 对比所有变体，选择路径最清晰、主题分区最明显的版本，然后基于它生成变体

**问题 B: 缺少响应式变体设计**
- 当前所有屏幕都是 2560×2048（桌面），但产品方案说"平板优先"
- **建议**: 至少为 Main Learning Screen 生成平板(1024×1366)和手机(390×844)的响应式版本

**问题 C: 缺少暗色模式**
- 家长可能让孩子在晚上使用
- **建议**: 设计系统支持暗色模式（护眼）

### 7. AI 导师的边界设计不足

**问题**: 产品方案没有定义 AI 导师的"安全护栏"
- 孩子可能问不适当的问题
- AI 可能给出不准确的技术建议
- **建议新增**:
  - AI 只能回答与当前课程相关的问题
  - 所有回复经过安全过滤层（类似 Code.org AI Tutor）
  - 当 AI 不确定时说"这个问题我也不太确定，要不要问问老师/家长？"
  - AI 使用次数限制（免费版每日5次需要明确提示）

## 🟢 锦上添花 (P2 — V2/V3 阶段)

### 8. 社区功能安全设计

**产品方案提到 "社区分享与展示"，但没有考虑儿童社区安全**:
- 不能让孩子直接接触陌生人
- 作品分享需要家长审批
- **建议**: 作品分享走"家长审核→生成链接"模式，而非开放社区

### 9. 离线模式

**产品方案 V3 提到 PWA 离线模式，但需要更早考虑**:
- 网络不好的地区（中国三四线城市、海外部分地区）
- **建议**: MVP 就用 Service Worker 缓存课程资源

### 10. 多语言支持

**产品方案目标是中国市场，但竞品分析全是英文产品**:
- 是否考虑出海？如果出海需要英文界面
- 如果不出海，中文界面的字体、排版需要专门优化（Quicksand 不支持中文）
- **建议**: 中文字体选择 — 推荐思源黑体(Noto Sans SC)或阿里巴巴普惠体，与 Quicksand 风格匹配

### 11. 定价策略需要再审视

| 当前方案 | 问题 | 建议 |
|---------|------|------|
| 免费15课 | 可能太少，用户还没体验核心价值就走了 | 免费25课（到第一个Turtle项目完成） |
| ¥29/月 | 对中国家庭合理，但需要对比竞品 | Tynker $9/月≈¥65，我们定价有优势 |
| 教育版¥999/年 | 太贵，学校采购流程复杂 | 改为按学生数计费：¥50/学生/年 |

## 📋 缺失屏幕清单 (需要补充的 Stitch 设计)

| 优先级 | 屏幕名称 | 说明 |
|--------|---------|------|
| P0 | **Onboarding Splash** | 吉祥物引入，60秒到第一课 |
| P0 | **Parent Dashboard** | 学习报告+屏幕时间+订阅管理 |
| P0 | **Error State** | 代码错误/网络错误/空状态 |
| P0 | **Loading Screen** | Python加载时的知识卡片 |
| P1 | **Registration (Parent)** | 家长注册流程（邮箱+密码+孩子信息） |
| P1 | **Registration (Child)** | 孩子选择头像+昵称（不收集真实信息） |
| P1 | **Settings** | 字体大小/语音开关/暗色模式 |
| P1 | **Main Learning — Tablet** | 平板响应式版本 |
| P1 | **Main Learning — Mobile** | 手机响应式版本 |
| P2 | **Project Gallery** | 作品展示墙（家长审核后） |
| P2 | **AI Tutor Chat** | AI对话全屏界面 |
| P2 | **Weekly Report** | 每周学习报告推送 |

## 🎯 总结: Top 5 立即行动项

1. **生成 Onboarding + 家长面板 两个 Stitch 屏幕** — 这是产品最关键的缺失
2. **修正 Level 1 课程结构** — Turtle 推迟到第11课，前10课纯概念
3. **选择中文字体** — Quicksand 不覆盖中文，需要搭配 Noto Sans SC
4. **定义 AI 安全护栏** — 范围限制 + 安全过滤 + 次数提示
5. **生成错误/空状态设计** — 吉祥物让错误变得友好
