> 完整产品方案文档 | 综合竞品分析与产品设计

## 一、竞品深度分析

### 1.1 竞品全景图

| 产品 | 核心优势 | 关键不足 | 可借鉴点 |
|------|----------|----------|----------|
| **Scratch** (MIT) | 1亿+用户，创意自由，零门槛 | 完全无法过渡到真实代码 | 社区文化、分享机制、积木式交互 |
| **Code.org** | 完整K-12课程，学校渗透率极高 | 积木→文本过渡突兀 | AI Tutor苏格拉底式引导、课程结构化 |
| **Tynker** ($9-20/月) | 积木→Python/JS平滑过渡，Minecraft联动 | 付费门槛高，免费内容有限 | 渐进式课程路径、项目制学习 |
| **CodeCombat** | RPG游戏化极强，2000万+玩家 | 直接跳到真实Python，对8岁太难 | 确定性模拟引擎、关卡式设计 |
| **EduBlocks** (Anaconda) | **每个积木=一行Python**，实时同步显示 | 界面简陋，缺乏游戏化 | Block→Text双向转换、文本在积木上显示 |
| **BlockPy/BlockMirror** | 三视图模式(积木/分屏/纯文本) | 面向数据科学，不适合儿童 | AST解析实现代码↔积木互转 |
| **KangaCode** | RPG冒险关卡，CSTA标准对齐 | 较新，生态不成熟 | 故事驱动关卡、AI调试助手 |
| **Wáàlá** | 手机优先，每天3分钟 | 内容较浅 | 碎片化学习、无笔记本也能用 |
| **Codewright** | AI导师、真实编辑器从第一天开始 | 对8岁可能太硬核 | AI导师引导式教学、工程思维培养 |
| **AI4Kids** | Think→Try→Reflect→Do四步学习法 | 较新平台 | 重视思考过程而非速度、标准化错误 |
| **Koda Academy** | Prompt→Generate→Understand→Break→Extend五步 | AI生成代码，不教写代码 | "Break It"故意留bug、理解优先 |
| **Pengi** (Scratch插件) | 语音交互、苏格拉底式引导 | 仅限Scratch | 语音交互对低龄儿童极其重要 |

### 1.2 竞品技术栈分析

| 技术层 | 方案 | 优势 | 劣势 | 推荐 |
|--------|------|------|------|------|
| **Python运行** | Skulpt (~250KB) | 轻量、启动快、有turtle | Python 2.x为主，Py3不完整 | ⭐ 推荐用于MVP |
| **Python运行** | Pyodide (~10MB) | 完整CPython、支持科学库 | 加载10+秒、内存占用大 | ⭐ 推荐用于完整版 |
| **Python运行** | Brython (~1.5MB) | 轻量 | 不支持标准库 | ❌ 不推荐 |
| **积木引擎** | Google Blockly | 行业标准、社区大 | 需要定制Python代码生成 | ⭐ 推荐 |
| **代码↔积木** | BlockMirror | 三视图、AST双向解析 | 依赖定制版Skulpt | ⭐ 推荐参考实现 |
| **代码编辑器** | Monaco Editor | VS Code同源、功能丰富 | 对儿童过于复杂 | ⭐ 可定制 |
| **代码编辑器** | CodeMirror | 轻量、Python语法高亮 | 不如Monaco强大 | 备选 |
| **游戏引擎** | Phaser.js | 2D游戏、Canvas渲染 | 需要额外学习 | 备选 |
| **可视化输出** | Turtle (Skulpt内置) | 零配置、Python标准库 | 功能有限 | ⭐ 推荐入门 |
| **可视化输出** | Canvas API | 灵活、强大 | 需要JS桥接 | ⭐ 推荐进阶 |

## 二、产品定位与核心价值

### 2.1 目标用户

| 维度 | 描述 |
|------|------|
| **主要用户** | 7-10岁儿童（零基础或有Scratch经验） |
| **辅助用户** | 家长（监控进度、亲子互动）、老师（班级管理） |
| **核心场景** | 碎片化学习（每天5-10分钟）、周末项目时间 |
| **设备优先** | 平板（iPad/Android） > 手机 > 电脑浏览器 |

### 2.2 一句话定位

> **"从积木到代码的彩虹桥"** — 让孩子在不知不觉中从拖拽积木过渡到写真实Python

### 2.3 四大核心差异点

| # | 差异点 | 竞品现状 | 我们的方案 |
|---|--------|----------|------------|
| 1 | **渐进式积木消退** | EduBlocks: 切换；Tynker: 模式切换 | 积木透明度逐渐降低，文字逐渐显现，最终只剩代码 |
| 2 | **即时可视化反馈** | CodeCombat: 角色移动；Skulpt: Turtle | 每个课程都有视觉输出（Turtle画布/小游戏/动画），不用控制台输出 |
| 3 | **AI导师用儿童语言** | Code.org AI Tutor: 苏格拉底式 | 错误解释用比喻："冒号就像开门，不开门代码进不去！" |
| 4 | **碎片化微课程** | Wáàlá: 3分钟 | 5分钟一课、进度自动保存、随时继续 |

### 3.2 核心模块详解

#### 模块A: BlockFade 渐进过渡引擎（核心创新）

这是产品最大的差异化点。灵感来自 EduBlocks + BlockMirror + CodeCombat 的渐进难度。

**四个阶段自动切换：**

```
Phase 1: 积木世界 (课程 1-10)
┌────────────────────────────────┐
│  [print "Hello!"]  [move 10]   │  ← 积木
│                                │
│  Python代码预览（只读）:        │
│  print("Hello!")               │  ← 看到真实代码
│  turtle.forward(10)            │
└────────────────────────────────┘

Phase 2: 混合世界 (课程 11-20)
┌────────────────────────────────┐
│  [print ___]  [move ___]       │  ← 积木变半透明
│      │            │            │
│  print("Hello!")               │  ← 可编辑真实代码
│  turtle.forward(10)            │
│                                │
│  ↑ 积木和代码双向联动            │
│  修改代码 → 积木更新             │
│  拖拽积木 → 代码更新             │
└────────────────────────────────┘

Phase 3: 代码世界 (课程 21-30)
┌────────────────────────────────┐
│  print(  )  ← 积木只剩提示      │
│  turtle.forward(  )           │
│                                │
│  print("Hello!")               │  ← 主要编辑区
│  turtle.forward(10)            │
│                                │
│  💡 提示: for循环可以重复...    │  ← AI提示
└────────────────────────────────┘

Phase 4: 纯代码 (课程 31+)
┌────────────────────────────────┐
│  print("Hello!")               │  ← VS Code风格编辑器
│  turtle.forward(10)            │
│  for i in range(5):            │
│      turtle.forward(20)        │
│                                │
│  🤖 问AI: "怎么写一个循环？"    │
└────────────────────────────────┘
```

#### 模块B: 可视化输出引擎

**核心原则：每个课程都有视觉反馈，不用无聊的控制台**

| 课程阶段 | 视觉输出 | 技术实现 |
|----------|----------|----------|
| 入门(1-10) | Turtle 画布画形状 | Skulpt内置turtle模块 |
| 基础(11-20) | 动画+小游戏 | Canvas API + JS桥接 |
| 进阶(21-30) | 交互式游戏 | Phaser.js 2D引擎 |
| 项目(31+) | 自定义项目 | Canvas/Phaser + 自由创作 |

**Turtle画布实现方案（MVP首选）：**
```python
# 孩子写的代码
import turtle
t = turtle.Turtle()
t.color("red")
for i in range(4):
    t.forward(100)
    t.right(90)

# 在浏览器中的呈现：
# ┌───────────────────────┐
# │      🐢 →───          │
# │          │            │
# │          ↓            │
# │      ◻️ 正方形         │
# └───────────────────────┘
```

#### 模块C: AI 儿童导师

**设计原则：苏格拉底式引导 + 儿童友好语言**

```python
# 系统 Prompt 设计
"""
你是一位耐心的编程老师，教7-10岁小朋友学Python。

规则：
1. 用比喻和故事解释概念，不用专业术语
2. 不要直接给答案，用问题引导孩子思考
3. 解释错误时用儿童能懂的语言
4. 每次回复不超过3句话
5. 用emoji增加亲和力

错误解释示例：
- 缺少冒号: "哎呀！Python的for循环需要一个冒号(:)来'开门'，
  没有冒号，循环里面的代码就进不去哦！"
- 缩进错误: "Python很在意对齐！就像排队一样，
  在for里面的代码需要多缩进4个空格，表示它们'属于'这个循环"
- 变量未定义: "Python说它不认识'name'这个小朋友。
  你是不是忘记先用 name = 'xxx' 来介绍它了？"
"""
```

#### 模块D: 课程引擎

**课程设计：100课渐进式路径**

```
Level 1: 积木启蒙 (课程 1-15)
  ├── 1-5: 打印魔法 (print, 字符串)
  ├── 6-10: 数字魔法 (变量, 数学运算)
  └── 11-15:  Turtle画第一笔 (导入库, 基本移动)

Level 2: 逻辑入门 (课程 16-35)
  ├── 16-20: 条件判断 (if/else) —  Turtle画彩色图形
  ├── 21-25: 循环基础 (for) — Turtle画多边形
  ├── 26-30: 循环进阶 (while, 嵌套) — Turtle画螺旋
  └── 31-35: 函数入门 (def) — Turtle画复杂图案

Level 3: 项目实战 (课程 36-65)
  ├── 36-40: 猜数字游戏
  ├── 41-45: 简易计算器
  ├── 46-50: 文字冒险游戏
  ├── 51-55: Turtle画迷宫
  ├── 56-60: Turtle小游戏 (接球/贪吃蛇)
  └── 61-65: 自由创作项目

Level 4: 高级探索 (课程 66-100)
  ├── 66-75: 列表与字典
  ├── 76-85: 面向对象入门
  ├── 86-95: 高级项目 (打砖块/飞机大战)
  └── 96-100: 毕业设计 — 完整游戏
```

**每课结构（5分钟完成）：**

```yaml
lesson:
  id: "lesson_006"
  title: "Turtle向前走"
  level: 1
  phase: "block"  # block | hybrid | hint | code
  
  # 学习目标
  objectives:
    - "学会 import turtle"
    - "使用 turtle.forward() 让海龟移动"
  
  # 故事引入 (30秒阅读)
  story: |
    有一只小海龟🐢，它想在沙滩上散步。
    你能告诉它走多远吗？
  
  # 积木配置 (Phase 1)
  blocks:
    - category: "turtle"
      available: ["import", "forward", "right", "left", "color"]
  
  # 预期代码
  expected_code: |
    import turtle
    t = turtle.Turtle()
    t.forward(100)
  
  # 视觉输出
  visual:
    type: "turtle_canvas"
    expected_result: "海龟向右移动100像素"
  
  # AI导师提示
  hints:
    - "试试把数字改成200，看看海龟走多远！"
    - "你知道t.right(90)会让海龟转什么方向吗？"
  
  # 验证规则
  validation:
    must_contain: ["import turtle", "forward"]
    run_test: true
    visual_check: "turtle_moved"
```

### 11.3 设计原则总结

1. **Tactile UI (触觉界面)**: 积木和按钮有"软弹"阴影，按下时缩小98%，提供即时触觉反馈
2. **Color Coding (色彩编码)**: 不同Python功能用不同颜色分类(循环=绿色、变量=蓝色、逻辑=紫色)
3. **Oversized Typography (超大字体)**: 所有文字比常规大20-30%，适应8岁儿童的阅读速度
4. **Extreme Roundness (极致圆角)**: 无任何尖角，最大化"儿童友好"感受
5. **High Contrast (高对比度)**: 白色容器配浅色边框，确保积木与静态信息清晰区分
